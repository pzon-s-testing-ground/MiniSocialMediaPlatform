# Security Policy

## Supported Versions

As an actively developing project, we currently only apply security updates to our most recent release.

| Version | Supported          |
| ------- | ------------------ |
| 1.0.x   | Yes                |
| < 1.0   | No                 |

## Reporting a Vulnerability

We take the security of this platform seriously. If you discover a security vulnerability within our React client or Node.js server, please do not disclose it publicly in the repository's issue tracker.

Instead, please report it directly to our development team by sending an email to [Insert Team/Security Email Here].

**When reporting a vulnerability, please include the following details:**

* A detailed description of the vulnerability and its potential impact on the system or users.
* Exact steps to reproduce the issue.
* Any relevant server error logs, client-side console outputs, or code snippets where the vulnerability occurs.

**What to expect:**

1. Our four-person development team will review the report and acknowledge receipt within 48 hours.
2. We will investigate the issue and determine a course of action.
3. If the vulnerability is confirmed, we will develop, review, and test a patch across our development branches.
4. We will notify you once the vulnerability has been resolved and the fix has been merged into the main branch.
